# Verse Roots

A Bible study app connecting English verses to their original Hebrew and Greek roots via Strong's concordance. Look up any verse, explore word-by-word original-language analysis, drill into lexicon definitions, and browse the concordance to find every occurrence of a root word across Scripture.

## Architecture

```
verse-roots/
├── apps/
│   ├── api/          — Express REST API (port 3001), serves verse/lexicon/concordance data
│   └── web/          — Vite + React + TypeScript frontend (port 5173)
├── packages/
│   └── db/           — Shared SQLite access layer (@verse-roots/db)
├── data/
│   ├── ingestion/    — Ingestion scripts (parse-stepbible.ts, parse-kjv.ts, etc.)
│   ├── db/           — SQLite database (gitignored, generated by ingestion)
│   └── raw/          — Downloaded raw data files (gitignored, cached locally)
└── LICENSES/         — Third-party license texts
```

## Running the Dev Server

```bash
# Install all workspace dependencies
npm install

# Start API + frontend concurrently
npm run dev
```

- Frontend: http://localhost:5173
- API: http://localhost:3001

## Re-running the Ingestion Pipeline from Scratch

```bash
# 1. Install workspace dependencies
npm install

# 2. Ingest STEPBible tagged texts + lexicons
npm run ingest

# 3. Ingest KJV translation text (public domain)
npm run ingest:kjv

# 4. Verify data integrity
npm run verify
```

The ingestion scripts download files from GitHub on first run and cache them in `data/raw/`. Subsequent runs skip the download.

## Phase 2: Accounts & Cloud Sync

### Prerequisites
- Supabase project (free tier works): https://supabase.com
- Stripe account: https://stripe.com

### Supabase setup
1. Create a new Supabase project
2. Run `supabase/migrations/001_initial.sql` in the SQL editor
3. Copy your project URL and anon key to `apps/web/.env.local`
4. Copy your service role key to `apps/api/.env`

### Stripe setup
1. Create a product "Verse Roots Pro" with a $7/mo recurring price
2. Copy the price ID and set it as `STRIPE_PRICE_ID` in `apps/api/.env`
3. Add a webhook endpoint pointing to `https://your-domain.com/api/stripe/webhook`
4. Subscribe to: customer.subscription.created/updated/deleted, checkout.session.completed
5. Copy the webhook secret to `apps/api/.env`

### Running locally with Phase 2
The app runs without any env vars — auth and sync simply won't activate.
Set the env vars above and restart to enable them.
